# R3mote

Ad-free, local-network mobile remote app for family use.

## What is implemented

- Cross-platform app with Expo Router + TypeScript
- TV discovery over local network (SSDP + mDNS)
- Remote control UI with large touch targets
- Pairing and persistence flows for:
  - Roku
  - Samsung
  - LG webOS
  - Sony Bravia
  - Vizio SmartCast
  - Android TV / Google TV
  - Fire TV (Android TV protocol fallback)
- Local storage of paired TVs and user preferences
- Internal distribution config for Android and iOS via EAS

## Prerequisites

- Node.js 18+
- Expo account
- EAS CLI account login
- Apple Developer account (for iOS installs on real devices)

## Install

```bash
npm install
```

## Development with Expo Dev Client

1. Build dev client once per platform:

```bash
eas build --platform android --profile preview
eas build --platform ios --profile preview
```

2. Install the generated dev client app on your phone.
3. Start Metro:

```bash
npx expo start --dev-client
```

4. Open the installed dev client and connect.

## Production/Internal Distribution

Build installable artifacts:

```bash
eas build --platform android --profile production
eas build --platform ios --profile production
```

### Android family install

- Share APK link/file to family.
- On each phone: allow unknown sources and install APK.

### iOS family install

- Register family device UDIDs in Apple Developer.
- Build with EAS iOS internal profile.
- Share install link generated by EAS.
- On each phone: trust the developer certificate in iOS settings.

## TV setup notes

- Roku: usually no setup required
- Samsung: approve prompt on TV on first connect
- LG: enable LG Connect Apps and approve prompt
- Sony: set IP Control and Pre-Shared Key
- Vizio: enter PIN shown on TV
- Android TV: enter pairing code shown on TV
- Fire TV: if discovery fails, add manually by IP and ensure network remote debugging options are enabled
